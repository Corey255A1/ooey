# Phase 4: Composable Scene Graph & Unified Input (Pending)

## Goal
Evolve the engine's core architecture from fixed-function rendering to a composable Retained Mode Scene Graph, and introduce a highly extensible, unified input system.

## Action Items

1. **Rendering Overhaul: The `Drawable` Primitive**
   - Deprecate `draw_rect`, `draw_line`, etc., directly on the `IRenderTarget`.
   - Introduce a base `Drawable` abstract class.
   - Implement low-level primitives: `RectPrimitive`, `CirclePrimitive`, `TextPrimitive`.
   - Define a mechanism for `IRenderTarget` to consume geometry (e.g., vertex and index buffers) generated by `Drawable` primitives, rather than knowing *what* shape it is rendering.

2. **UI Composability**
   - Create a `View` base class that acts as a container for `Drawable` components.
   - Example: A `Button` class that is constructed by layering a `RectPrimitive` (background) and a `TextPrimitive` (label).
   - Establish a rendering queue/Z-index ordering system to draw stacked objects correctly.

3. **Unified Abstract Input System (Completed)**
   - Define a universal `Pointer` struct: `{ id, x, y, state }`.
   - Implement an `InputManager` capable of tracking an arbitrary number of active pointers. This abstraction must not care if the pointer is driven by a physical mouse, a touchscreen finger, or a game controller.
   - Define an `IInputProvider` interface that platforms (like `X11WindowBackend` or a hypothetical `SerialPortInputProvider`) implement to push hardware data into the `InputManager`.
   - Define a similar abstraction for keyboard input (`KeyEvent`).

4. **Integration with the Backend (Completed)**
   - Update `X11WindowBackend` to push mouse movement and clicks into the new `InputManager`.
   - Refactor `OpenGLRenderTarget` to consume and render the new `IDrawable` batch system.

5. **Interaction Model & The Controller**
   - Create an `IInteractive` interface requiring bounding box/hit-testing functions and event handler hooks.
   - Implement a basic `Controller` to poll `InputManager`.
   - Implement top-down geometric pointer routing (hit-testing). If a `Pointer` is over a stack of objects, the top-most `IInteractive` object consumes the event.
   - Implement state-based focus routing for `KeyEvent`s, routing raw keyboard events directly to the actively focused UI element.
